Main Building
  • Switch off all unnecessary lights in the base camp buildings

Control Room
  • Turn on all camera fans: on arrayctl: start_camera_fan -a (details)
  • Check that the CFD thresholds and Lookback Times are correct:

    • ssh vdaq@dacq.t1
    • src/checkCFD - this gives for each telescope the median and lists the lowest and highest 6 values.
    • Alternatively, try:
      ssh vdaq@dacq.t1
      vdbget_CFDSettings -t# (where #=1,2,3,4) threshold should be ~45mV,
      vdbget_FADCChanSettings -t# (where #=1,2,3,4) lookback time should be 3000 samples (details)

NO LATER THAN 75 MINUTES BEFORE STARTUP

Control Room: Control02 Computer
  • Check that ReadTemps is running on all 4 telescopes by opening readtemps desktop and if necessary clicking on RT icon
    (or Log into arrayctl and launch FADC temperature monitors: ssh arrayctl, startReadTemps.pl -1234) (details)
  • Launch DACQ. (This turns on the FADCs, so chillers must be on first): ssh arrayctl, startVDAQ.pl -1234 (details)
  • Execute grb_play to test that the GRB alert sound is working (details)


Control Room: Control04 Computer
  • Launch Tracking: click the TRK icon (or startTRK.pl –a), check to see that telescopes are in Array Mode. (details)
  • Launch VAC: click the VAC icon (or ssh arrayctl, VAC &) (details)
  • Check for recent GRB alerts on the GRB panel of array_tracking (details)
  • After a few minutes check VAC to see that event builders are connected
  • In VAC, select Observer->Start Night
  • Check that VAC has connected to harvester, database and L3. L3 should be in the initialized state. (details here and here)
  • Check that sufficient disk space is free (details) In VAC: - Observer->Check Free Disk Space
Control Room: Control01 Computer
  • Launch the Infrared Radiometers: click on the FIR Icon (or “RUNFIR &); WAIT a minute for windows to appear (details)
  • Launch the All-Sky Cloud Monitor by clicking the All-Seeing Eye Icon (or execute, vncviewer 10.0.0.143) (details). Click on "Start Video", set an exposure time of ~10 sec.
  • Launch the lidar http://lidar.vts (bookmarked). Right click on tab, "Reload Every" -> "Enable" and choose appropriate time interval.
  • Launch the array monitor http://arraymon.vts (bookmarked as VTSSlowCtl)



L2 Trigger GUI
  • On control01 start the L2 trigger GUI using the L2 icon or by typing: java -jar /home/observer/L2Software/L2TriggerGUI.jar
  • Click the "Connect to CORBA Servers" button in the "Monitoring" tab.
  • In the "Configuration" tab of the GUI, check that all pixels in each telescope are either inverted and enabled or un-inverted and disabled. This can be done by clicking the "Refresh All Pixels" button. All pixels should be either green or black. There should be no pixels that are inverted and disabled (blue).
  • Check that there are no stuck on pixels by clicking the "Check for triggered/stuck pixels" button in the "Configuration" tab. Any pixels that show up as red should be disabled and un-inverted.
    Note: If the FADCs are not powered on, all pixels will appear red.
  • In the "Configuration" tab check that the detune is on (green color on the circular mark besides of on/off button), and the L2 Output width is set to 5.
  • In the "Monitoring" tab, from the "All" view, click the "Read Timing Alignment Info" button. Select "Delays" in the drop-down menu. Ensure that delays are loaded (i.e., the pixels have non-zero delay values.) for each telescope. For more information see Timing Alignment Verification

Control Room
  • After the preamps are started, launch Current Monitor: click on the IMON icon (or startIMon.pl -1234) (details)
  • Once Imon is started, launch HV: click the HV icon (or execute startHV.pl -1234a) (details)
  • Turn the HV on (PMT->All On), current monitors should read ~0.
  • Take a 2-minute fake flasher run:

    • Start the flasher ( ssh arrayctl ; $ flasher )
    • Start a standard flasher run in VAC and make sure all sub-systems are working fine
    • Monitor the data being taken with quicklook: ssh harvester, then run ql_monitor.
  • If there's a Star Party or other activity outside the gate, read Overview of Observing. Talk to people/hand out flyers if necessary.

STARTUP

  • Final check: outside - is the sun down / basecamp lights out?
  • Slew to zenith (85º), then start slewing to the first source of the night. You want to be on target and tracking before astronomical twilight.
  • Check that the 2 main radios in the Control Room are respectively on the "Normal" and "VERITAS" bands
  • Make sure the volumes are set appropriately (we turn it down during the day)
  • Announce over the radio that VERITAS is turning on
  • Turn the HV on in hvac (PMT->All On), watch the current monitors.
  • During first run, perform a During Night checklist

Each telescope trailer has a thermostat to set the temperature. Check the paper on the wall of each trailer to see what temperature setting to use. In all telescopes, the thermostat is on the wall to the right (or left) of the chiller control box at the very back of the trailer. The thermostats have a rotating dial to set the temperature. Make sure to set the switch on cool (in some trailers a snowflake may appear on the display after a few seconds).
Each telescope has a dehumidifier behind the FADC racks. Check to see that it is running. The dehumidifier in each telescope has a drain which dumps the collected water to the outside, so you don't have to empty these anymore.
